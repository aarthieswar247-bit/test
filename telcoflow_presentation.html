03 / PROBLEM

What Breaks Today

🔴 Manual, Unstructured Intake

Deviation requests arrive via email, WhatsApp, and verbal escalations — no standard format, no policy check, no audit trail.

🔴 Policy Interpretation Is Guesswork

Approvers rely on memory and experience. No one has time to read 200-page TRAI circulars for every billing credit request.

🔴 Approval Chains Break Down

Mandatory steps are skipped. Finance approves without Operations sign-off. Compliance reviews happen after execution — or not at all.

🔴 Zero Audit Readiness

When TRAI asks for records, teams scramble across inboxes and spreadsheets. Audit prep takes weeks. Penalties follow.

Structured deviation intake with auto-validation against policy thresholds
AI-generated, policy-grounded justifications — never fabricated
Role-based, sequential approvals that cannot be bypassed
AI advisory at each approval step with role-specific recommendations
Immutable audit log — regulator-ready in seconds, not weeks

07 / WORKFLOW DEEP DIVE

How State Flows Through TelcoFlow

DEVIATION LIFECYCLE STATES

PENDING Submitted, awaiting guardrail evaluation
BLOCKED Guardrail failed — hard policy ceiling exceeded
ANALYSED AI justification generated, approval chain opened
IN REVIEW Active human approvals in sequential chain
APPROVED All 4 approval steps passed — execution authorised
REJECTED Any approver rejected — workflow terminated, audit written

DEVIATION TYPES + POLICY THRESHOLDS

08 / PROMPTING DISCIPLINE

Role–Action–Output (RAO) Patterns

RAO STRUCTURE
ROLE "You are a compliance documentation specialist for a Telecom/Media company."
ACTION "Write formal, professional business justifications referencing ONLY policy clauses from the provided context."
OUTPUT "3 paragraphs + 'Policy References:' line. Under 300 words. Formal register."
GUARDRAILS IN PROMPTING
temperature=0.2 — deterministic, formal output
Grounding instruction: "Never fabricate policy clauses"
Role-specific advisor prompts per approval step
Max tokens constrained (800 justify / 300 advise)
Hard policy context injected — model cannot deviate
OBLIGATION EXTRACTOR — STRICT JSON PROMPT
// System prompt for structured policy extraction { "role": "system", "content": "You are a regulatory obligation extractor for TRAI/ITU policy documents. OUTPUT FORMAT: Return ONLY valid JSON. No preamble. No markdown fences. No explanation. Schema: { 'obligation_id': string, // e.g. TRAI-BC-2.3.1 'obligation_text': string, // verbatim clause text 'deviation_types': [string], // applicable deviation types 'threshold_value': number|null,// numeric limit if stated 'threshold_unit': string|null, // INR / days / percent 'severity': 'hard'|'soft', // hard = absolute limit 'effective_date': string|null // ISO 8601 } If a field cannot be determined, use null. Never invent obligations not present in the text." }
